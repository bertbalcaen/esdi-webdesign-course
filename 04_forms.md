    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-client-vs-server" class="anchor" aria-hidden="true" href="#client-vs-server"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Client vs server</h1>
<ul>
<li>Forms are one of the main ingredients of web applications.</li>
<li>Web applications are split up into 2 parts: the client and the server.</li>
</ul>
<table>
<thead>
<tr>
<th></th>
<th>Client</th>
<th>Server</th>
</tr>
</thead>
<tbody>
<tr>
<td>Where?</td>
<td>Here, on the user's computer</td>
<td>There, on a computer somewhere else in the world, in a datacenter</td>
</tr>
<tr>
<td>Technology</td>
<td>HTML, CSS, JS</td>
<td>PHP, Java, JS, Ruby, ...</td>
</tr>
</tbody>
</table>
<p>Information can flow in both directions:</p>
<ul>
<li>From server to client: when you open a link to a webpage in your browser, the server sends back the HTML, CSS, images, ...</li>
<li>From client to server: when you submit a form, you the send the data from your browser to the server.</li>
</ul>
<h1><a id="user-content-an-example" class="anchor" aria-hidden="true" href="#an-example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>An example</h1>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">h1</span><span class="pl-kos">&gt;</span>Contact<span class="pl-kos">&lt;/</span><span class="pl-ent">h1</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">form</span> <span class="pl-c1">action</span>="<span class="pl-s">process_contact_form.php</span>" <span class="pl-c1">method</span>="<span class="pl-s">post</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span>
        <span class="pl-kos">&lt;</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
            Your name
            <span class="pl-kos">&lt;</span><span class="pl-ent">br</span><span class="pl-kos">&gt;</span>
            <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">your_name</span>"<span class="pl-kos">&gt;</span>
        <span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;/</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span>
        <span class="pl-kos">&lt;</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
            Message
            <span class="pl-kos">&lt;</span><span class="pl-ent">br</span><span class="pl-kos">&gt;</span>
            <span class="pl-kos">&lt;</span><span class="pl-ent">textarea</span> <span class="pl-c1">name</span>="<span class="pl-s">message</span>"<span class="pl-kos">&gt;</span><span class="pl-kos">&lt;/</span><span class="pl-ent">textarea</span><span class="pl-kos">&gt;</span>
        <span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;/</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span>
        <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">submit</span>" <span class="pl-c1">value</span>="<span class="pl-s">Send</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;/</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">form</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>HTML is used to create form elements.</li>
<li>There are different types of form elements: single line text fields, multi-line textfields, buttons, ...</li>
<li>The form elements must be inside a <code>&lt;form&gt;</code> element. It serves as a container for the form.</li>
</ul>
<h1><a id="user-content-processing-forms" class="anchor" aria-hidden="true" href="#processing-forms"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Processing forms</h1>
<ul>
<li>HTML is a markup language, not a programming language.</li>
<li>You can use it to give meaning to documents, but it doesn't have programming logic.</li>
<li>To process a form, you need a programming language.</li>
<li>'Processing a form' could mean: emailing the form data, storing them in database, doing a search in a database, ...</li>
<li>Examples of common programming languages used on the web: PHP, Ruby, JavaScript, Java, ...</li>
<li>In most cases, the form is processed on the server, not inside the browser itself.</li>
</ul>
<p>Here is a simple example of a PHP script that can email form data from the example:</p>
<div class="highlight highlight-text-html-php"><pre><span class="pl-ent">&lt;?php</span>
<span class="pl-en">mail</span>(<span class="pl-s">'bert@rekall.be'</span>, <span class="pl-s">'Message from '</span> . <span class="pl-s1"><span class="pl-c1">$</span><span class="pl-c1">_POST</span></span>[<span class="pl-s">'your_name'</span>], <span class="pl-s1"><span class="pl-c1">$</span><span class="pl-c1">_POST</span></span>[<span class="pl-s">'message'</span>]);
<span class="pl-en">header</span>(<span class="pl-s">'Location: thanks.html'</span>);
exit;</pre></div>
<ul>
<li>You would need to save this as <code>process_contact_form.php</code>, and place it on a server that can handle PHP scripts.</li>
<li>When the user clicks the submit button, the data is sent to <code>process_contact_form.php</code>.</li>
<li>The variables <code>$_POST['your_name']</code> and <code>$_POST['message']</code> will be replaced with the actual data that the user entered. Note that these correspond to the <code>name</code> attributes of the form elements.</li>
<li>After sending the mail, the user will be redirected to <code>thanks.html</code>.</li>
</ul>
<h1><a id="user-content-form-validation" class="anchor" aria-hidden="true" href="#form-validation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Form validation</h1>
<p>This is a simplified example. In a real situation, you would make sure that the <code>your_name</code> and <code>message</code> fields are not empty, and remove any type of input that could be dangerous.</p>
<p>This is called 'form validation'. Examples are:</p>
<ul>
<li>Making sure that a text field contains an email address.</li>
<li>Removing dangerous content from form data, to prevent hacking.</li>
<li>Requiring that certain fields are not empty.</li>
<li>...</li>
</ul>
<p>Validation can be done in 2 places:</p>
<ul>
<li>On the 'client', meaning in the browser, so using HTML en JavaScript.</li>
<li>On the server, so in PHP or whatever programming language you are using.</li>
</ul>
<p>Both have advantages and disadvantages:</p>
<ul>
<li>Client-side validation is quick. There is no need to send the data to the server first. Server-side validation by definition requires sending the data to the server first and examining it there. This is an extra step.</li>
<li>As designers and developers, we never have full control over what happens on the client-side. Users can change HTML, disable JavaScript, ... This is not the case with server-side validation. A user can't change the way our server-side logic works.</li>
</ul>
<p>A good website or application will do both:</p>
<ul>
<li>Client-side validation because it's quick and interactive.</li>
<li>Server-side validation because it guarantees clean data in the app.</li>
</ul>
<h1><a id="user-content-the-form-element" class="anchor" aria-hidden="true" href="#the-form-element"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>&lt;form&gt;</code> element</h1>
<p>Basic structure:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">form</span><span class="pl-kos">&gt;</span>
    <span class="pl-c">&lt;!-- form elements here --&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">form</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li><code>&lt;form&gt;</code> is a container for form elements.</li>
<li>An HTML page can have multiple forms.</li>
</ul>
<p>A <code>&lt;form&gt;</code> element has some important attributes.</p>
<h2><a id="user-content-the-action-attribute" class="anchor" aria-hidden="true" href="#the-action-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>action</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">form</span> <span class="pl-c1">action</span>="<span class="pl-s">process_contact_form.php</span>"<span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>The value of the <code>action</code> attribute is a URL.</li>
<li>When the user submits the form, the data will be sent to that URL. The form data will be available to the program at that URL.</li>
<li>The program can then process the data. See the example above, with the PHP script that mails the form data.</li>
<li>If you omit this attribute, then the form data will be submitted to the current page. This can be useful if you have a form inside a PHP page for example.</li>
</ul>
<h2><a id="user-content-the-method-attribute" class="anchor" aria-hidden="true" href="#the-method-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>method</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">form</span> <span class="pl-c1">method</span>="<span class="pl-s">post</span>"<span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>There are only 2 possible values for the <code>method</code> attribute: <code>get</code> (the default) and <code>post</code>.</li>
<li>If you omit this attribute, then <code>get</code> will be used.</li>
<li>If submitting the form will have side-effects, then you should use <code>post</code>.</li>
<li>A side-effect could be: something inside the database is changed (the name of a person is updated), an email is sent, a picture is deleted, ...</li>
<li>If there are no side effects, then use <code>get</code>. Examples: doing a search, ...</li>
</ul>
<p>When using <code>get</code>, the data will be visible in the URL as key/value pairs after submitting the URL. The advantage is that the user can bookmark the result of the form submission.</p>
<p>Here's a simple search form:</p>
<div class="highlight highlight-text-html-basic"><pre> <span class="pl-kos">&lt;</span><span class="pl-ent">form</span> <span class="pl-c1">action</span>="<span class="pl-s">search.php</span>" <span class="pl-c1">method</span>="<span class="pl-s">get</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">search</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">submit</span>" <span class="pl-c1">value</span>="<span class="pl-s">Search!</span>"<span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">form</span><span class="pl-kos">&gt;</span></pre></div>
<p>After submitting the form, the URL in the browser's address bar will be:</p>
<pre><code>search.php?search=ESDI
</code></pre>
<p>When using <code>post</code>, the form data is not visible in the URL, so it can't be bookmarked.</p>
<h1><a id="user-content-general-info-about-form-elements" class="anchor" aria-hidden="true" href="#general-info-about-form-elements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>General info about form elements</h1>
<ul>
<li>There are many different types of form elements.</li>
<li>They all have characteristics in common, so let's look at these first.</li>
</ul>
<h2><a id="user-content-the-name-attribute" class="anchor" aria-hidden="true" href="#the-name-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>name</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>"<span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Most form elements require a <code>name</code> attribute.</li>
<li>If you don't supply a value for it, then the form element will not be sent when the form is submitted.</li>
<li>The value is important when processing the form. In the PHP for example, the value of the element above would be available as <code>$_GET['first_name']</code> or <code>$_POST['first_name']</code>.</li>
</ul>
<h2><a id="user-content-the-value-attribute" class="anchor" aria-hidden="true" href="#the-value-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>value</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">value</span>="<span class="pl-s">Bert</span>"<span class="pl-kos">&gt;</span></pre></div>
<p>Used to set the initial value of text fields. For other field types (checkboxes, radio buttons, ...) this will usually correspond to the id of an option in the database.</p>
<h2><a id="user-content-the-placeholder-attribute" class="anchor" aria-hidden="true" href="#the-placeholder-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>placeholder</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">placeholder</span>="<span class="pl-s">Type your first name here, for example Bert</span>"<span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Works for text fields.</li>
<li>Used to explain what kind data is expected.</li>
<li>Will be removed once the user starts entering data.</li>
</ul>
<h2><a id="user-content-the-required-attribute" class="anchor" aria-hidden="true" href="#the-required-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>required</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">required</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Used to mark a field as required.</li>
<li>The browser will not let you submit the form if the field is empty.</li>
<li>Note that this is client-side validation, to it's very easy to circumvent. Never rely on this if it's important that the field is really required.</li>
<li>You should always visually indicate which fields are required. The convention is to add an asterisk in front of the label, for example <code>* First name</code>.</li>
</ul>
<h2><a id="user-content-the-readonly-attribute" class="anchor" aria-hidden="true" href="#the-readonly-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>readonly</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">readonly</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Used to make an element read-only. It cannot be changed.</li>
<li>The data is still sent to the server.</li>
</ul>
<h2><a id="user-content-the-disabled-attribute" class="anchor" aria-hidden="true" href="#the-disabled-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>disabled</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">disabled</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Used to make an element read-only. It cannot be changed.</li>
<li>The data will not be sent to the server.</li>
</ul>
<h2><a id="user-content-the-maxlength-attribute" class="anchor" aria-hidden="true" href="#the-maxlength-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>maxlength</code> attribute</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">maxlength</span>="<span class="pl-s">255</span>"<span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Used to set the maximum allowed length of the data.</li>
<li>Again, this is client-side validation, so it's not safe to rely on this.</li>
</ul>
<h2><a id="user-content-the-autofocus-attribute" class="anchor" aria-hidden="true" href="#the-autofocus-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>autofocus</code> attribute</h2>

<li>Used to automatically set the focus on a field when the page is loading.</li>
<li>Be careful with this because it might confuse the user.</li>

<h2><a id="user-content-the-label-element" class="anchor" aria-hidden="true" href="#the-label-element"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The <code>&lt;label&gt;</code> element</h2>
<ul>
<li>HTML is all about semantics, about using the correct elements to indicate what meaning each part has in your document.</li>
<li>Form element need labels (descriptions) so that users know what to enter where.</li>
<li>These descriptions should be inside <code>&lt;label&gt;</code> elements.</li>
<li>By default, browsers don't make a visual distinction for <code>&lt;label&gt;</code>s. They look like regular text.</li>
<li>But: when a user clicks on a label, the focus is placed inside the corresponding form element. It also helps visually impaired people to understand how they have to complete a form.</li>
</ul>
<p>A common usage is to wrap the description + the form element itself inside a <code>&lt;label&gt;</code> element. Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
    First name
    <span class="pl-kos">&lt;</span><span class="pl-ent">br</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>"<span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span></pre></div>
<p>You are not required to wrap the form element inside the <code>&lt;label&gt;</code>. Both can be separate, and then you can use the <code>for</code> attribute of the <code>&lt;label&gt;</code> to indicate to which element it belongs. In that case, you must give the form element an <code>id</code> attribute, and use its value as the value for the <code>&lt;label&gt;</code>'s <code>for</code> attribute.</p>
<p>For example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">label</span> <span class="pl-c1">for</span>="<span class="pl-s">first_name</span>"<span class="pl-kos">&gt;</span>
        First name
    <span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">text</span>" <span class="pl-c1">name</span>="<span class="pl-s">first_name</span>" <span class="pl-c1">id</span>="<span class="pl-s">first_name</span>"<span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span></pre></div>

        <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">checkbox</span>" <span class="pl-c1">name</span>="<span class="pl-s">colors[]</span>" <span class="pl-c1">value</span>="<span class="pl-s">red</span>"<span class="pl-kos">&gt;</span> Red
    <span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">br</span><span class="pl-kos">&gt;</span>

    <span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">br</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
        <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">checkbox</span>" <span class="pl-c1">name</span>="<span class="pl-s">colors[]</span>" <span class="pl-c1">value</span>="<span class="pl-s">blue</span>"<span class="pl-kos">&gt;</span> Blue
    <span class="pl-kos">&lt;/</span><span class="pl-ent">label</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>Use checkboxes when you have multiple options, and multiple options can be selected. Use a radio button group when only 1 option can be selected.</li>
<li>Use the same <code>name</code> for checkboxes belonging to the same group. Quite often, you will see that the <code>name</code> ends with <code>[]</code>. This is related with the processing of the data on the server.</li>
<li>Just like with radio buttons, you can use the <code>checked</code> attribute to check a checkbox.</li>
<li>The <code>value</code> of the selected checkboxes will be sent to the server.</li>
<li>If you have more than a few options, a <code>&lt;select&gt;</code> can be more appropriate. See later on.</li>
<li>You can change the way check boxes look with CSS, but be careful, because users are used to the default look.</li>
</ul>
<h2><a id="user-content-a-dropdown-or-select-list" class="anchor" aria-hidden="true" href="#a-dropdown-or-select-list"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>A dropdown or select list</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">select</span> <span class="pl-c1">name</span>="<span class="pl-s">country</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">option</span> <span class="pl-c1">value</span>=""<span class="pl-kos">&gt;</span>Choose<span class="pl-kos">&lt;/</span><span class="pl-ent">option</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">option</span> <span class="pl-c1">value</span>="<span class="pl-s">ES</span>"<span class="pl-kos">&gt;</span>Spain<span class="pl-kos">&lt;/</span><span class="pl-ent">option</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">option</span> <span class="pl-c1">value</span>="<span class="pl-s">BE</span>"<span class="pl-kos">&gt;</span>Belgium<span class="pl-kos">&lt;/</span><span class="pl-ent">option</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">select</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>A <code>&lt;select&gt;</code> will create a dropdown list.</li>
<li>Inside the element, create one <code>&lt;option&gt;</code> for every option.</li>
<li>The text for the option goes inside the <code>&lt;option&gt;</code> element.</li>
<li>Just like with checkboxes and radio buttons, the <code>value</code> of the selected <code>&lt;option&gt;</code> will be sent to the sever when the form is submitted. It's often a database id.</li>
<li>Usually, the first <code>&lt;option&gt;</code> is an 'empty' option, with a short description, like 'Choose', or 'Choose your country'. This is done because otherwise the first 'actual' option would be automatically selected.</li>
<li>Add the <code>selected</code> attribute to make an <code>&lt;option&gt;</code> selected. (Note: not <code>checked</code> like with checkboxes and radio buttons.)</li>
</ul>
<p>By default, only 1 option can be selected. So: it has the same functionality as a radio button group. The advantage of course is that a <code>&lt;select&gt;</code> uses much less space. The disadvantage is: the user has to click on the list to see the available options.</p>
<ul>
<li>When there are few options, a radio group is usually more user friendly.</li>
<li>If there are many options (for example all countries of the world), then a <code>&lt;select&gt;</code> is better.</li>
</ul>
<p>A select can also support multiple selections, so then it becomes similar to a group of checkboxes. To do this, add a <code>multiple</code> attribute:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">select</span> <span class="pl-c1">name</span>="<span class="pl-s">country[]</span>" <span class="pl-c1">multiple</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">option</span> <span class="pl-c1">value</span>="<span class="pl-s">ES</span>"<span class="pl-kos">&gt;</span>Spain<span class="pl-kos">&lt;/</span><span class="pl-ent">option</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">option</span> <span class="pl-c1">value</span>="<span class="pl-s">BE</span>"<span class="pl-kos">&gt;</span>Belgium<span class="pl-kos">&lt;/</span><span class="pl-ent">option</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">select</span><span class="pl-kos">&gt;</span></pre></div>
<p>Multiple options can be selected using Control-click. Most users don't know this, so it's best to mention this. If you only have a couple of options, and you need multi-select, then a group of checkboxes is more user friendly.</p>
<h2><a id="user-content-a-file-field" class="anchor" aria-hidden="true" href="#a-file-field"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>A file field</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">form</span> <span class="pl-c1">enctype</span>="<span class="pl-s">multipart/form-data</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">file</span>" <span class="pl-c1">name</span>="<span class="pl-s">profile_picture</span>"<span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">form</span><span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>This will show a 'Browse ...' button.</li>
<li>You should add the attribute <code>enctype="multipart/form-data"</code> to the <code>&lt;form&gt;</code> element if you want to enable file uploads.</li>
</ul>
<h2><a id="user-content-a-submit-button" class="anchor" aria-hidden="true" href="#a-submit-button"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>A submit button</h2>
<p>Example:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">input</span> <span class="pl-c1">type</span>="<span class="pl-s">submit</span>" <span class="pl-c1">value</span>="<span class="pl-s">Save</span>"<span class="pl-kos">&gt;</span></pre></div>
<ul>
<li>When the user clicks on the button, the form will be submitted.</li>
<li>The button text is set with the <code>value</code> attribute.</li>
</ul>
<p>There is another way to create a submit button:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">button</span> <span class="pl-c1">type</span>="<span class="pl-s">submit</span>"<span class="pl-kos">&gt;</span>Save<span class="pl-kos">&lt;/</span><span class="pl-ent">button</span><span class="pl-kos">&gt;</span></pre></div>
<h1><a id="user-content-exercise" class="anchor" aria-hidden="true" href="#exercise"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Exercise</h1>
<p>Create a form for a user account page:</p>
<ul>
<li>Be sure to use <code>&lt;label&gt;</code>'s and <code>&lt;fieldsets&gt;</code>'s where appropriate.</li>
<li>Don't forget the <code>&lt;form&gt;</code> element, and the <code>method</code>.</li>
<li>Also visually indicate where the required fields are.</li>
</ul>
<p>It should be able to handle this info:</p>
<ul>
<li>First and last name: required. Make sure your name is displayed in the fields.</li>
<li>Email: required. Show your email in the field.</li>
<li>Year of birth: required</li>
<li>Gender: required. Options: female, male, other, and 'I'd rather not say'. Only 1 option allowed. Make sure 1 is selected.</li>
<li>Country: required. Only 1 option allowed. The app only works in Spain, Portugal, France, Greece and Belgium, so only these can be chosen. Make sure 1 is selected.</li>
<li>Website: optional. Add the placeholder text 'Must start with http:// or https://'.</li>
<li>Biography: optional. Multiple lines of text. Make sure it contains some lines of text.</li>
<li>Interests: optional. Multiple options possible. Options: architecture, design, music, photography, fashion. Make sure 1 or 2 are selected.</li>
<li>Password: optional.</li>
<li>Profile picture: optional.</li>
<li>Buttons to save.</li>
</ul>
<p>Validate the HTML using <a href="https://validator.w3.org/#validate_by_input" rel="nofollow">https://validator.w3.org/#validate_by_input</a>.</p>
